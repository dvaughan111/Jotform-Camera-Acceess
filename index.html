Why This Works for JotForm PDFs

The key to displaying images as thumbnails in JotForm PDFs is using the correct data structure when submitting the form data.

The Problem

When you submit [object Object] to JotForm, it can't interpret the image data correctly for PDF generation.

The Solution

We need to provide JotForm with a properly formatted data structure that includes:

  • The image URL or base64 data
  • Filename for the image
  • PDF display options as a separate property

PDF Display Options

{ display: 'thumbnail', width: 'auto', height: 'auto', alignment: 'center', border: '1px solid #ccc', padding: '5px' }

Data Submission Format

Each image should be submitted with this structure:

{ url: 'data:image/jpeg;base64,...', fileName: 'photo-123456789.jpg', pdfOptions: { ... } }

Testing the Solution

After implementing this fix:

  1. Take a photo using the widget
  2. Approve the photo
  3. Click "Done"
  4. Submit your JotForm
  5. Check the PDF output - images should now display as thumbnails
